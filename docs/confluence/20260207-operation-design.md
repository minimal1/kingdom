## 핵심 능력 3가지

### 1. 유지보수성 (Maintainability)
- 꾸준히 일하는 게 가장 중요
- 하드웨어 리소스 모니터링 + 적절한 조치

### 2. 기록성 (Record-keeping)
- 어떤 상황인지, 어떤 작업을 얼마 동안 수행했는지 기록
- 문제 해결 장치 + 개선 포인트 확인 장치

### 3. 작업 수행 능력 (Task Execution)
- 핵심 기능, 가장 복잡한 구성

## 역할 체계 (계층 구조)

| 역할 | 비유 | 책임 |
|------|------|------|
| 파수꾼 (Sentinel) | 나라를 지키는 파수꾼 | 티켓, GitHub PR, PR 코멘트 등 데이터 모니터링 → 왕에게 전달 |
| 왕 (King) | 왕 | 데이터를 받아 판단, 전문성에 맞는 장군에게 지시 |
| 장군 (General) | 각 장군 | 명령 수행을 위해 병사들에게 구체적 업무 분배 |
| 병사 (Soldier) | 병사 | PR Review, Test Code 작성, Jira 티켓 처리 등 실제 작업 |
| 사절단 (Envoy) | 사절단 | Slack 통해 사람과 소통, 상황 공유/응답 전달 |

## 작업 수행 흐름
1. 모니터링 → 작업 선택
2. 작업 여력 확보
3. 작업 이해
4. 필요한 작업 분할
5. 각 작업 진행 (연관 작업은 대기)
6. 진행상황 체크 + 공유

## 과제
- 메모리 문제: CC의 컨텍스트 제한 → 효과적인 메모리 관리법 필요
- 대시보드: 리소스 상태 + 진행중인 작업 한눈에 보기
