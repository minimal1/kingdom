🔄 END-TO-END DATA FLOW
1 파수꾼이 GitHub/Jira 변화 감지 → event.json 생성
2 왕이 이벤트 소비 → 리소스/충돌 확인 → 적합한 장군에게 task.json 배정
3 장군이 도메인 메모리 로딩 → 프롬프트 조립 → claude -p로 병사 생성
4 병사가 실제 작업 수행 (코드 리뷰/테스트/구현) → result.json 저장
5 장군이 task 완료 확인 → 성공 시 왕에게 보고, 실패 시 재시도 (max 2)
6 왕이 결과 확인 → 사절에게 Slack 알림 요청
7 내관이 전체 시스템 건강 상태를 지속 모니터링 → 왕에게 리소스 정보 제공
