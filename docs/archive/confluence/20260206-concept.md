# 20260206 - Lil Eddy 만들기

> Source: https://querypie.atlassian.net/wiki/x/AQDIcQ

## 배경
- ClaudeCode로 개발하는게 일상화됨
- friday(PR 리뷰), saturday(리팩토링), sunday(Jira 티켓) 플러그인 운영 중
- 체감 결과물은 기대 대비 60~70% → 사람 터치 필요
- 주니어 수준엔 충분하고 더 빠르게 끝남

## 프로젝트 컨셉

| 구성요소 | 매핑 |
|---------|------|
| 지능 | LLM + Prompt |
| 기억 | Memory 관리 |
| 몸 | 하드웨어 리소스 (EC2) |
| 손발 | CLI, MCP 등 도구 |

## 핵심 목표
- EC2에 로컬과 같은 업무 환경 구성
- 주 작업들을 자동화
- 쉴새없이 일하며, 간단한 업무를 사람 개입 없이 진행

## 오케스트라 구성
- 특정 작업에 특화된 작업자
- 작업자들을 통솔하고 진행을 조율하는 관리자
- Tmux-Orchestrator 차용: Tmux + 시스템 루프로 ClaudeCode 간 소통

## 주요 작업 (우선순위)
1. PR Review ← 기존 friday 플러그인 활용
2. Test Code 작성
3. Jira Ticket 대응 ← 기존 sunday 플러그인 활용
4. Code Refactoring ← 리스크 높아 후순위

## 필요한 구현 블록
- 기본적인 도구들
- ClaudeCode Plugins
- 메모리 관리 방식
- 오케스트라 구성
- 외부 이벤트 캐치 방식
- 이벤트 정의 및 로깅
- 리소스 모니터링
- 리소스 기반 행동 규칙
- 사람과 소통 방식
